01 // OVERVIEW
GAME CONCEPT

XENUS_SETTLEMENT adalah mini playable demo yang di-embed langsung di landing page xenor.xyz. Player membangun settlement kecil di atas grid deterministik — menempatkan bangunan, melihat fase eksekusi berjalan secara real-time, dan membuktikan bahwa seed yang sama selalu menghasilkan simulasi yang identik. Ini bukan game terpisah — ini adalah live showcase dari xenor-xenus itu sendiri.

02 // MECHANICS
CORE GAME LOOP
Mechanic 01
SEED INPUT

Player input seed string (u64 atau text → hashed ke u64). Seed menentukan terrain awal, resource distribution, dan event schedule secara deterministik. Share seed = share world yang sama persis.

Mechanic 02
GRID PLACEMENT

Click-to-place bangunan di grid 12×12. Setiap cell memiliki terrain type (dari seed). Collision detection otomatis. Building adjacency memberikan bonus resource — mirror dari sistem di xenor-xenus.

Mechanic 03
PHASE TICK

Setiap N detik, satu siklus phase dijalankan: RESOURCE → PRODUCTION → POPULATION → EVENT → SNAPSHOT. Player bisa lihat tiap phase dalam progress bar realtime. Ini adalah xenor-xenus phase system secara live.

Mechanic 04
RESOURCE FLOW

3 resource utama: ENERGY, NODES, TOKENS. Tiap bangunan produce/consume resource berbeda. Balance = settlement tumbuh. Imbalance = degradation. Visual flow arrows antar bangunan.

Mechanic 05
DETERMINISM PROOF

Di corner layar: replay button yang reset simulasi ke tick 0 dengan seed yang sama. Hasilnya selalu identik — ini adalah demo live dari deterministic execution xenor protocol.

Mechanic 06
PROTOCOL UNLOCK

Saat settlement mencapai threshold tertentu, animasi "PROTOCOL_ACTIVE" muncul dan reveal CTA: "Explore full xenor-xenus →". Langsung convert curiosity jadi user acquisition.

03 // PHASE SYSTEM
EXECUTION CYCLE

Phase system di game langsung mirror dari xenor-xenus phase trait. Setiap tick cycle menjalankan 5 phase berurutan — deterministik, tidak bisa di-skip, urutan tidak bisa diubah. Ini adalah visualisasi live dari Phase::execute() di Rust backend.

01
RESOURCE
Hitung resource production dari semua bangunan aktif
02
PRODUCTION
Apply building output ke settlement resource pool
03
POPULATION
Update population count berdasarkan housing + food
04
EVENT
Jalankan seed-driven event (attack, discovery, bonus)
05
SNAPSHOT
Serialize state ke deterministic snapshot untuk replay
// xenor-xenus phase execution — ditampilkan live di game UI pub trait Phase { fn execute(&self, state: &mut GameState) -> PhaseResult; fn name(&self) -> &str; fn order(&self) -> u32; // determinism-critical: must never change } // Game UI polls phase progress dan animasikan tiap phase saat execute pub fn run_cycle(state: &mut GameState, phases: &[Box<dyn Phase>]) { for phase in phases.iter() { state.current_phase = phase.name().to_string(); phase.execute(state); // UI di-notify setiap step ini } state.tick += 1; state.snapshot(); // deterministic snapshot untuk replay }

// DETERMINISM NOTE

Semua building stats di-define sebagai pure functions — tidak ada randomness di production phase. Randomness hanya di EVENT phase, dan itu pun seed-driven. Artinya: replay dengan seed yang sama = hasil identik sampai tick manapun.

05 // TECH STACK
IMPLEMENTATION STACK
Layer Technology Peran dalam Game
CORE ENGINE Rust + xenor-xenus Game logic, phase execution, deterministic simulation — reuse langsung dari repo xenor-xenus
WASM BRIDGE wasm-bindgen + wasm-pack Compile Rust ke .wasm, expose GameState, run_cycle(), dan get_snapshot() ke JavaScript
RENDERER React + TypeScript Grid renderer, building placement UI, resource bars, phase progress indicator, seed input form
GRAPHICS HTML5 Canvas / CSS Grid Game grid pakai CSS Grid untuk simplicity. Animasi phase tick pakai Canvas overlay untuk particle effects
STATE SYNC SharedArrayBuffer / JSON WASM memory ↔ React state bridge. GameState di-serialize ke JSON setiap snapshot untuk React re-render
SEED HASHING FNV-1a (pure Rust) Convert text seed input → u64 hash deterministik. FNV-1a ringan, zero-dependency, consistent cross-platform
BUILD Vite + wasm plugin Bundle Rust WASM + React TS jadi single optimized bundle. Target: <500KB gzipped untuk fast landing page load
HOSTING Vercel / Cloudflare Pages Static hosting, edge caching. WASM file served dengan correct Content-Type dan CORS headers
ANALYTICS Plausible (optional) Track: seeds yang paling sering dipakai, average session time, protocol_unlock conversion rate

// SEED VISUALIZER UX

Seed input field pakai style terminal: blinking cursor, typewriter animation saat user input. Submit → grid animate "GENERATING FROM SEED..." dengan progress bar deterministik, lalu bangunan spawn satu per satu sesuai seed. Kalau dua tab dibuka dengan seed sama → hasilnya pixel-perfect identik. Ini adalah WOW moment yang demonstrate core value protokol.
